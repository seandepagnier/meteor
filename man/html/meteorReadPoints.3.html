NAME

meteorReadPoints, meteorReadTriangles, meteorWritePoints, meteorWriteTriangles  

SYNOPSIS

#include <meteor.h>

int meteorReadPoints(int count, int format, int type, void *data);
int meteorReadTriangles(int count, int format, int type, void *data);
int meteorWritePoints(int count, int format, int type, const void *data);
int meteorWriteTriangles(int count, int format, int type, const void *data);  

DESCRIPTION

These functions provide low-level access to point and triangle data. The orientation of the triangle is determined by the order of the triple.  

COUNT

The count specifies how many Points or Triangles to process with this call.  

FORMAT

The format parameter determines what data to operate on. The format is either METEOR_INDEX, or a bitwise mask of METEOR_COORDS, METEOR_NORMALS, METEOR_COLORS, and METEOR_TEXCOORDS. The pointdata that is stored or retrieved from the data parameter is either a single value which is an index for the point, or a sequence of tightly packed triples containing at least one of: coords, normals, colors, or texture coordinates in this order. All data has 3 parts except index data has 1 part. An index of 0 is the first point when using meteorReadPoints or meteorWritePoints. The highest index is 1 less than meteorPointCount(). METEOR_INDEX cannot be used when writing points. When writing points, a new point is created if METEOR_COORDS is specified, otherwise, the current point's data is updated. When writing triangles, a new triangle is always created.  

TYPE

The type parameter specifies the type of the data, METEOR_INT, METEOR_UNSIGNED_INT, METEOR_FLOAT, METEOR_DOUBLE, and METEOR_LONG_DOUBLE are supported.  

RETURN VALUE

On error, -1 is returned, and meteorError is set. Otherwise the count processed is returned. When reading, if this value is less than count then there is no more data to read. Subsequent calls before calling meteorRewind will return 0.  

ERRORS

These functions will fail if:
1. The data format requested does not exist in the meteor.
2. The data type is not supported.
3. When reading data, meteorRewind was not called since other meteor modification has occured.
4. When writing triangles, specifying data along with index data.
